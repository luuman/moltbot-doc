Moltbot on DigitalOcean

目标​

在 DigitalOcean 上以 $6/月 (或预留定价 $4/月) 运行一个持久的 Moltbot 网关。

如果您想要 $0/月的选项并且不介意 ARM + 供应商特定的设置，请参阅 Oracle Cloud 指南。

成本对比 (2026)​

提供商套餐规格价格/月备注
Oracle CloudAlways Free ARM最多 4 OCPU, 24GB RAM$0ARM, 有限容量 / 注册怪癖
HetznerCX222 vCPU, 4GB RAM€3.79 (~$4)最便宜的付费选项
DigitalOceanBasic1 vCPU, 1GB RAM$6简单 UI, 好文档
VultrCloud Compute1 vCPU, 1GB RAM$6多个位置

选择提供商:

  • DigitalOcean: 最简单的用户体验 + 可预测的设置 (本指南)
  • Hetzner: 良好的性价比 (参见 Hetzner 指南)
  • Oracle Cloud: 可以是 $0/月, 但更繁琐且仅 ARM (参见 Oracle 指南)

先决条件​

  • DigitalOcean 账户 (使用 $200 免费积分注册)
  • SSH 密钥对 (或愿意使用密码认证)
  • ~20 分钟

1) 创建 Droplet​

  1. 登录 DigitalOcean
  2. 点击 创建 → Droplets
  3. 选择:
    • 区域: 距离您最近的 (或您的用户)
    • 镜像: Ubuntu 24.04 LTS
    • 大小: Basic → Regular → $6/月 (1 vCPU, 1GB RAM, 25GB SSD)
    • 认证: SSH 密钥 (推荐) 或密码
  4. 点击 创建 Droplet
  5. 记下 IP 地址

2) 通过 SSH 连接​

ssh root@YOUR_DROPLET_IP

3) 安装 Moltbot​

# 更新系统
apt update && apt upgrade -y

# 安装 Node.js 22
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs

# 安装 Moltbot
curl -fsSL https://molt.bot/install.sh | bash

# 验证
moltbot --version

4) 运行入门向导​

moltbot onboard --install-daemon

向导将引导您完成:

  • 模型认证 (API 密钥或 OAuth)
  • 通道设置 (Telegram, WhatsApp, Discord, 等)
  • 网关令牌 (自动生成)
  • 守护进程安装 (systemd)

5) 验证网关​

# 检查状态
moltbot status

# 检查服务
systemctl --user status moltbot-gateway.service

# 查看日志
journalctl --user -u moltbot-gateway.service -f

6) 访问仪表板​

网关默认绑定到回环。要访问控制 UI:

选项 A: SSH 隧道 (推荐)

# 从您的本地机器
ssh -L 18789:localhost:18789 root@YOUR_DROPLET_IP

# 然后打开: http://localhost:18789

选项 B: Tailscale Serve (HTTPS, 回环专用)

# 在 droplet 上
curl -fsSL https://tailscale.com/install.sh | sh
tailscale up

# 配置网关使用 Tailscale Serve
moltbot config set gateway.tailscale.mode serve
moltbot gateway restart

打开: https://<magicdns>/

说明:

  • Serve 使网关保持回环专用，并通过 Tailscale 身份头进行认证。
  • 要改为要求令牌/密码，请设置 gateway.auth.allowTailscale: false 或使用 gateway.auth.mode: "password"。

选项 C: Tailnet 绑定 (无 Serve)

moltbot config set gateway.bind tailnet
moltbot gateway restart

打开: http://<tailscale-ip>:18789 (需要令牌)。

7) 连接您的通道​

Telegram​

moltbot pairing list telegram
moltbot pairing approve telegram <CODE>

WhatsApp​

moltbot channels login whatsapp
# 扫描 QR 码

参见 通道 了解其他提供商。

1GB RAM 的优化​

$6 的 droplet 只有 1GB RAM。为了让一切运行顺畅:

添加交换分区 (推荐)​

fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab

使用较轻的模型​

如果您遇到内存不足，考虑:

  • 使用基于 API 的模型 (Claude, GPT) 而不是本地模型
  • 将 agents.defaults.model.primary 设置为较小的模型

监控内存​

free -h
htop

持久性​

所有状态都存在于:

  • ~/.clawdbot/ — 配置、凭据、会话数据
  • ~/clawd/ — 工作区 (SOUL.md, memory, 等)

这些在重启后仍然存在。定期备份:

tar -czvf moltbot-backup.tar.gz ~/.clawdbot ~/clawd

Oracle Cloud 免费替代方案​

Oracle Cloud 提供 Always Free ARM 实例，比此处的任何付费选项都强大得多 —— $0/月。

您得到的规格
4 OCPUsARM Ampere A1
24GB RAM绰绰有余
200GB 存储块卷
永久免费无信用卡费用

注意事项:

  • 注册可能很麻烦 (如果失败请重试)
  • ARM 架构 — 大多数东西都能工作，但某些二进制文件需要 ARM 构建

有关完整设置指南，请参见 Oracle Cloud。有关注册技巧和故障排除注册过程，请参见此 社区指南。

故障排除​

网关无法启动​

moltbot gateway status
moltbot doctor --non-interactive
journalctl -u moltbot --no-pager -n 50

端口已被占用​

lsof -i :18789
kill <PID>
